Die 3-2-1-Regel gilt auch für Selbsthoster

3 Kopien der Daten, auf 2 verschiedenen Medien, davon 1 an einem anderen Ort.
Wer nur eine lokale Sicherung hat, hat kein Backup. Er hat eine zweite Quelle für denselben Hausbrand.

Open Source Desktop System-Snapshots

ℹ️
Proprietär, aber lokal

Veeam Agent for Linux Free ist closed-source. Der kritische Unterschied zu Cloud-Zwangs-Produkten: Die Sicherung läuft vollständig lokal oder auf deinen eigenen Zielen — NFS, SMB, externes Medium. Keine Daten gehen zu Veeam.

Debian / Ubuntu

bash · Repo & Installation 
# GPG-Key importieren
curl -fsSL https://www.veeam.com/downloads/veeam-release-key.pub \
  | gpg --dearmor \
  | sudo tee /usr/share/keyrings/veeam-archive-keyring.gpg > /dev/null

# Repository eintragen (Debian 12 / Ubuntu 22.04+)
echo "deb [signed-by=/usr/share/keyrings/veeam-archive-keyring.gpg] \
  https://repository.veeam.com/backup/linux/agent/dpkg/debian/x86_64 bookworm veeam" \
  | sudo tee /etc/apt/sources.list.d/veeam.list

sudo apt update && sudo apt install veeam

RPM-Systeme (RHEL / Rocky / Fedora)

bash 
sudo rpm --import https://www.veeam.com/downloads/veeam-release-key.pub
sudo dnf install https://repository.veeam.com/backup/linux/agent/rpm/rhel/x86_64/veeam-release-rhel-1.0-1.x86_64.rpm
sudo dnf install veeam

Starten

bash 
# Terminal-Oberfläche (TUI) starten
sudo veeam

Linux Mint / Ubuntu Desktop

Backup-Job anlegen (TUI)

  1. TUI starten

    sudo veeam → im Menü „Configure Backup" auswählen

  2. Backup-Modus wählen

    Entire Machine für Bare-Metal-Recovery, Volume Level für einzelne Partitionen, File Level für Verzeichnisse

  3. Backup-Ziel festlegen

    Lokales Verzeichnis, NFS-Mount, SMB-Share oder Veeam Backup & Replication Server

  4. Zeitplan konfigurieren

    Täglich, wöchentlich oder manuell. Inkrementelle Sicherungen werden automatisch erstellt.

  5. Job ausführen & Logs prüfen

    Ersten Lauf manuell anstoßen, Ausgabe kontrollieren.

NFS-Ziel vorbereiten (empfohlen)

bash · NFS-Mount einrichten 
sudo apt install nfs-common
sudo mkdir -p /mnt/backup-nfs

# Test-Mount
sudo mount -t nfs 192.168.1.10:/volume1/backups /mnt/backup-nfs

# Dauerhaft in /etc/fstab
echo "192.168.1.10:/volume1/backups /mnt/backup-nfs nfs defaults,_netdev 0 0" \
  | sudo tee -a /etc/fstab

Job per CLI anlegen

bash · Entire Machine → NFS 
sudo veeamconfig job create entirenmachine \
  --name "Server-Vollbackup" \
  --repoName "NAS-Backup" \
  --compressionLevel 5

# Job manuell starten
sudo veeamconfig job start --name "Server-Vollbackup"

# Status prüfen
sudo veeamconfig session list
🔥
Recovery-ISO jetzt erstellen, nicht nach dem Crash

Wer erst nach dem Systemausfall an die Recovery-Medien denkt, hat verloren. ISO erstellen, booten, testen — das kostet 20 Minuten und rettet im Ernstfall Stunden.

bash 
sudo veeamconfig livecd create \
  --output /tmp/veeam-recovery.iso

# Auf USB-Stick schreiben (Gerät anpassen!)
sudo dd if=/tmp/veeam-recovery.iso of=/dev/sdX bs=4M status=progress
sync

Die ISO enthält minimalen Linux-Kernel + Veeam Recovery-Umgebung inkl. Netzwerktreiber.

USB-Stick beim Boot auswählen (BIOS/UEFI → Boot-Reihenfolge). Die Recovery-Umgebung startet automatisch im TUI-Modus und fragt nach dem Backup-Repository. NFS-Mount wird unterstützt.

  1. Repository verbinden

    NFS-Share oder lokales Laufwerk mit den Backups angeben

  2. Restore-Punkt wählen

    Datum/Uhrzeit der gewünschten Sicherung auswählen

  3. Ziel-Laufwerk auswählen

    Originalplatte oder neue Hardware — Veeam passt Partitionsgrößen an

  4. Wiederherstellung starten

    Je nach Datenmenge und Netzwerk 30 bis 90 Minuten

  5. Bootloader neu schreiben

    GRUB-Neuinstallation anbieten lassen — bei neuer Hardware immer wählen

bash · File-Level-Restore auf laufendem System 
sudo veeamconfig backup mount \
  --backupId <id> \
  --mountDir /mnt/veeam-restore

# Dateien manuell kopieren
cp /mnt/veeam-restore/home/user/dokument.pdf ~/

# Mount freigeben
sudo veeamconfig backup unmount --mountDir /mnt/veeam-restore

Systemd-Timer (empfohlen statt Cron)

/etc/systemd/system/veeam-backup.service 
[Unit]
Description=Veeam Backup Job
After=network-online.target
Wants=network-online.target

[Service]
Type=oneshot
ExecStart=/bin/bash -c 'veeamconfig job start --name "Server-Vollbackup"'
User=root
/etc/systemd/system/veeam-backup.timer 
[Unit]
Description=Tägliches Veeam Backup

[Timer]
OnCalendar=*-*-* 02:30:00
RandomizedDelaySec=300
Persistent=true

[Install]
WantedBy=timers.target
bash · Timer aktivieren 
sudo systemctl daemon-reload
sudo systemctl enable --now veeam-backup.timer
sudo systemctl list-timers veeam*

Monitoring-Skript

bash · /usr/local/bin/check-veeam.sh 
#!/bin/bash
STATUS=$(sudo veeamconfig session list | tail -1 | awk '{print $NF}')
if [[ "$STATUS" != "Success" ]]; then
  echo "Veeam Backup fehlgeschlagen: $STATUS" \
    | mail -s "[ALARM] Backup-Fehler $(hostname)" admin@example.com
fi

Tipp: Healthchecks.io selbst hosten

Healthchecks.io ist open source und per Docker deploybar. Backup-Skript sendet HTTP-Ping nach Erfolg — bleibt er aus, kommt die Alarm-Mail. Kein Drittdienst nötig.

bash · Timeshift — installieren & Snapshot erstellen 
sudo apt install timeshift

# Snapshot manuell erstellen (vor apt upgrade)
sudo timeshift --create --comments "Vor System-Update $(date +%Y-%m-%d)"

# Snapshots anzeigen
sudo timeshift --list

# Zu Snapshot zurückrollen
sudo timeshift --restore --snapshot "2026-03-19_02-00-01"
bash · BorgBackup — Repository initialisieren & Backup erstellen 
sudo apt install borgbackup

# Repository auf Remote-Server initialisieren (verschlüsselt)
borg init --encryption=repokey user@backup-server:/backups/mein-repo

# Backup erstellen
borg create --stats --progress \
  user@backup-server:/backups/mein-repo::{hostname}-{now:%Y-%m-%d} \
  /home /etc /var/www \
  --exclude /home/*/.cache

# Alte Backups rotieren (7 tägl., 4 wöchentl., 6 monatl.)
borg prune -d 7 -w 4 -m 6 user@backup-server:/backups/mein-repo

borgmatic — YAML statt Bash-Skript

borgmatic kapselt Borg-Aufrufe in eine übersichtliche YAML-Konfiguration und übernimmt Zeitplan, Prune-Logik und Monitoring. Installierbar via pip install borgmatic.
